MicroPython — it's the standard for Raspberry Pi Pico projects like this.
It's Python, so you already know the syntax from Boot.dev. The main difference is you're running it on bare hardware with no OS underneath, so instead of importing regular Python libraries you'll be using micro-versions designed for microcontrollers.
The two libraries you'll lean on most:

machine — controls GPIO pins, SPI bus (how you talk to the APA102 LEDs), and timers
utime — microsecond-level timing, critical for POV sync

The rough shape of the firmware logic will be:
1. Hall sensor fires → record timestamp (this = 0°)
2. Calculate RPM from time between pulses
3. Divide 360° into columns (one per LED refresh cycle)
4. Loop: at each column angle → push the right pixel column to the LED strip via SPI
5. Repeat ~30x per rotation for a full image
The APA102's SPI interface is actually one of the reasons we picked it over WS2812B — MicroPython's SPI is fast enough to update all 39 LEDs between column steps at normal riding speed.
You'll write and flash code from your MacBook using Thonny (free IDE built for MicroPython on Pico) or just VS Code with the MicroPico extension.
